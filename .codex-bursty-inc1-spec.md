## Routing policy (`internal/policy`) — the "same shape as TR" contract
Peek the JSON body ONCE (decode into `struct{ Model string; Provider
*struct{ Only []string; Order []string } ; ...raw json.RawMessage }`-style
minimal view + keep the raw bytes for forwarding; never re-encode the body
on passthrough paths).
Decision for /v1/chat/completions (and future local-capable endpoints):
1. `provider.only == ["local"]` (exactly) → LOCAL, forced (no burst); strip
   the `provider` field from the forwarded body (splice raw JSON, do not
   re-marshal the whole body — implement a small helper that removes a
   top-level key from raw JSON bytes and test it).
2. `provider.only`/`provider.order` mentioning anything other than "local" →
   TRUSTEDROUTER, forced; body forwarded VERBATIM.
3. Model prefix `local/<name>` → LOCAL, forced; forward with model rewritten
   to `<name>` (raw-splice the model value) and `provider` stripped.
4. Otherwise (default policy): LOCAL-FIRST — try to acquire a semaphore slot
   (buffered channel, cap local-max-concurrency) waiting up to
   local-queue-wait; on success forward to local; on no slot → BURST to TR.
   With burst-on-error: local connect error/timeout, HTTP 429/5xx, or 404
   whose body mentions the model (ollama "model not found") → burst to TR —
   ONLY if zero response bytes were already written to the client (buffer
   the local response until status+first body byte are known).
No local-url configured → all → TR. No TR key → all → LOCAL (429 when full).

## Proxy (`internal/proxy`, `internal/upstream`)
- LOCAL upstream: plain net/http with a pooled Transport; path = localBase +
  "/v1/chat/completions" (config note: if -local-url already ends in /v1,
  don't double it — normalize). Streaming: io.Copy with http.Flusher flush
  per write (wrap ResponseWriter), passing through SSE bytes untouched.
- TR upstream: the SDK client (Options{APIKey, BaseURL}); use RawRequest
  with the VERBATIM raw body so every TR arg passes through; stream the
  response body to the client the same flushing way. TR auth uses the
  configured key — the inbound Authorization header is NEVER forwarded
  upstream (it's Bursty's own -token, if set).
- Responses: always set `X-Bursty-Route: local|trustedrouter` and
  `X-Bursty-Reason: forced|policy|burst-full|burst-error` headers. For
  non-streaming JSON responses, additionally inject a top-level
  `"bursty":{"route":…,"reason":…}` object (raw-splice before the final
  `}`; tested). Streaming bodies are never modified.
- GET /v1/models: merged catalog — TR models via the SDK (cache 60s,
  serve-stale-on-error) + local models from localBase `/v1/models` (ollama
  supports it; 5s timeout; on error just omit), local entries duplicated as
  both bare id and `local/<id>` with `"owned_by":"local"`. Local-only or
  TR-only modes degrade gracefully.
- GET /healthz → {"ok":true,"local":bool,"trustedrouter":bool} (liveness of
  config, not upstream probes). GET /stats → JSON counters (expvar-backed):
  in_flight_local, bursts_full, bursts_error, forced_local, forced_tr,
  requests_total, per-route counts.
- Inbound auth: when -token set, every /v1/* request requires
  `Authorization: Bearer <token>` (constant-time compare); /healthz stays
  open.
- Errors use the TR error envelope shape:
  `{"error":{"code":…,"message":…,"type":…,"source":"bursty"}}`.
- Performance discipline: no full-body JSON re-marshal on any forward path;
  peek-decode only; reuse buffers where natural (bufio); server has sane
  ReadHeaderTimeout; upstream contexts inherit the inbound request ctx.

## Tests (httptest fake local + fake TR; -race clean; table-driven)
1. Directive matrix: provider.only ["local"] → local + provider stripped;
   provider.order ["anthropic"] → TR verbatim (assert body bytes identical);
   model "local/llama3" → local with model "llama3"; default → local.
2. Burst-on-full: cap=1, first request parked in the fake local (blocked
   channel), second request → TR; X-Bursty-Reason=burst-full; semaphore
   released after completion (third request → local again). Race test.
3. Burst-on-error: local connect refused → TR; local 500 → TR; local 404
   model-not-found body → TR; burst-on-error=false → error surfaces as-is.
4. Zero-bytes-sent guard: fake local that writes 200 + half an SSE stream
   then dies → response is NOT retried to TR (client got bytes); assert no
   TR call.
5. Streaming passthrough: SSE chunks from fake local arrive incrementally
   (flusher recorded), bytes verbatim; same for fake TR.
6. Local-only mode full → 429 + Retry-After + TR-envelope error body.
7. Models merge: fake TR catalog + fake local /v1/models → merged, prefixed,
   cache behavior (second call within TTL hits cache — count fake hits).
8. bursty JSON block injected on non-streaming; absent on streaming.
9. Token auth: 401 without/with-wrong token; /healthz open.
10. Raw-splice helpers: property-style table tests (nested "provider" keys
    untouched, string escapes, model value replacement).
