## 1. Decision must carry BOTH bodies (policy.go + proxy.go) — structural

Today `Decide` returns one `Body`, raw on the default-local path (provider
unstripped → leaks TR-shaping args like provider:{"sort":"price"} to ollama)
— and if we stripped it there, the BURST path would forward the stripped body
to TR, losing the user's provider args. Fix:
- `Decision{Route, Reason, LocalBody, TRBody []byte, View}` — TRBody is
  ALWAYS the original raw bytes (verbatim passthrough); LocalBody is ALWAYS
  provider-stripped (+ model rewritten when `local/`-prefixed), computed
  lazily/only when the route can hit local.
- Every local forward uses LocalBody (forced AND default-policy paths);
  every TR forward — forced, burst-full, burst-error — uses TRBody.
- Tests: default-policy local send has provider stripped; the SAME request
  bursting (full and error) reaches the fake TR with the ORIGINAL body bytes
  including the provider field, byte-identical to what the client sent.

## 2. All-local `only` directive must force local (policy.go)

`provider.only=["local","local"]` (or any all-"local" non-empty list)
currently falls through to burstable default policy, violating the user's
only-restriction. Replace `isExactlyLocalOnly` with: Only is non-empty AND
every entry == "local" → forced local (no burst). Keep order=["local"] as
non-forcing (order expresses preference, not restriction — add that comment).
Test both.

## 3. Release the local slot before bursting (proxy.go)

`defer s.releaseLocalSlot()` holds the local concurrency slot for the entire
TR burst response on the burst-on-error path — a slow TR response shrinks
local capacity. Restructure: release the slot at the moment local is
abandoned (before serveTrustedRouterChat on the error-burst path) with a
release-once guard so the defer stays safe. Test: cap=1, request A gets a
local error and bursts to a SLOW fake TR (blocked); while A's burst is still
streaming, request B must acquire the local slot and be served locally.

## 4. Multi-value inbound headers (upstream.go, nit)

`sanitizedExtraHeaders` forwards only values[0]; join multiple values with
", " per RFC 9110 so e.g. multiple Accept values survive.
