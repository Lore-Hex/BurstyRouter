## 1. Remaining endpoints (mirror the chat handler patterns exactly)
- `POST /v1/embeddings`: LOCAL-CAPABLE (ollama serves /v1/embeddings) — same
  policy engine (directives, local-first, semaphore, burst rules, dual
  bodies), same annotations. shouldBurst 404-model logic applies.

## 2. The `bursty-setup` skill (required deliverable)
`skills/bursty-setup/SKILL.md` — a Claude-Code-compatible agent skill (YAML
frontmatter: name: bursty-setup, description triggering on "set up
burstyrouter", "wire bursty into cursor", "expose my local model", etc.) that
interactively walks a user through:
1. Verify/run BurstyRouter (install via `go install
   github.com/Lore-Hex/BurstyRouter/cmd/burstyrouter@latest` or a release
   binary; minimal invocation lines for: ollama local-only; local+TR burst;
   TR-only) and verification curls (/healthz, a models call, one chat call
   with X-Bursty-Route asserted in the response headers).
2. ngrok exposure when their agent harness is remote: authtoken setup,
   `ngrok http 8383`, stable domains, and the STRONG instruction to set
   BURSTY_TOKEN when internet-exposed (with the curl -H "Authorization:
   Bearer" verification).
3. Harness wiring — the skill should ASK which harness and emit only that
   path, with copy-paste blocks + a verification step each:
   - Cursor: Settings → Models → OpenAI API override (base URL =
     https://<host>/v1, key = BURSTY_TOKEN or any string when unset), model
     entries like `local/llama3.2` or `anthropic/claude-haiku-4.5`.
   - Claude Code / Anthropic SDKs: ANTHROPIC_BASE_URL=https://<host>
     (Bursty's /v1/messages TR passthrough), ANTHROPIC_API_KEY as above.
   - aider / OpenAI SDKs / Codex CLI / OpenHands: OPENAI_BASE_URL (or
     OPENAI_API_BASE) = https://<host>/v1, OPENAI_API_KEY as above.
   - Generic: any OpenAI-compatible integration → point it at /v1.
4. Routing cheatsheet: how to pin (`local/` prefix, provider.only), how
   bursting behaves, reading X-Bursty-Route/X-Bursty-Reason and /stats.
Mirror the content as human docs in `docs/SETUP.md` (skill = interactive
flow; SETUP.md = linear reference). Keep both tight and copy-paste-first.
