# Task: BurstyRouter Increment F — slow-local hedge (burst when local first-token blows a deadline)

On the committed tree. Do NOT commit. Do not touch `.codex-*`.

Existing burst triggers: no local slot (full), local connect error, local
429/5xx, local 404-model. F adds a fourth: local "technically working but too
slow to first token" (cold model / huge context).

## Config
`-local-slow-after <duration>` (env BURSTY_LOCAL_SLOW_AFTER, default 0 =
disabled). A local request that has not produced its FIRST response BYTE
within the deadline is abandoned and bursts to cloud — subject to the SAME
rules as other bursts: only when not forced, cloud available, -cloud mode
allows, budget not exhausted, and ZERO bytes already sent to the client.

## Implementation
- New `policy.ReasonBurstSlow` in the Reason consts.
- In serveLocalChat/serveLocalCapable, after sending to local: read headers,
  then race the FIRST BODY BYTE vs the -local-slow-after timer (watch the
  BODY, not headers — Ollama/llama.cpp return 200 headers instantly but the
  first token slowly). Timer fires first → cancel the local ctx,
  releaseLocalSlot() (release-once, before the burst), reuse the existing
  burst path with ReasonBurstSlow and the ORIGINAL body (TRBody). First byte
  arrives in time → stream that already-read byte then the rest (do not lose
  it). Non-streaming: same first-byte deadline (~= response start).
- Forced-local (local/ prefix, provider.only local) does NOT hedge — waits
  (debug log if it exceeds the deadline).
- Stats counter bursts_slow; per-burst log reason=burst-slow. If /metrics or
  /ui files exist in the tree (parallel increments — may not), add the "slow"
  reason label; guard by checking, do not fail if absent.

## Tests
Unit: the first-byte-deadline race helper in isolation (fake ReadCloser
delaying the first byte → timer fires → "slow" signal; fast first byte →
byte + reader intact); forced-local does not hedge. Functional/e2e
(self-skip in sandbox): fake local sends 200 headers then stalls N ms before
first SSE byte, -local-slow-after < N, default request → X-Bursty-Reason:
burst-slow, reaches fake TR, client got ZERO local bytes; fast local → served
local, full body; forced-local + slow → waits, served local. Ollama-gated:
real request, tiny -local-slow-after (1ms) + TR configured → bursts.
