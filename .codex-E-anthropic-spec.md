## New package `internal/anthropic`
### Request translation (Anthropic → OpenAI), scope v1 = text + tools + streaming
- `system` (string OR content-block array) → leading `messages[0]` role
  system (concatenate text blocks).
- messages: role user/assistant; content string OR block array:
  - text blocks → string content (join). 
  - tool_use blocks (assistant) → OpenAI `tool_calls`
    [{id, type:"function", function:{name, arguments: json-string}}].
  - tool_result blocks (user) → role:"tool" messages {tool_call_id, content
    (stringified — text blocks joined; is_error noted by prefixing
    "[error] ")}. A user message mixing tool_result + text splits into the
    tool messages followed by a user message.
  - image blocks: v1 UNSUPPORTED → 400 invalid_request_error with a clear
    message ("images not supported on the local path; remove or route to
    cloud with provider.order").
- `tools` → OpenAI tools [{type:"function", function:{name, description,
  parameters: input_schema}}]. `tool_choice`: {type:"auto"}→"auto",
  {"any"}→"required", {"tool",name}→{"type":"function","function":{name}}.
- max_tokens→max_tokens (REQUIRED by Anthropic shape; pass through),
  temperature/top_p/stop_sequences→stop, stream→stream. Ignore/drop with a
  debug log: anthropic_version, metadata, cache_control hints, thinking
  budgets (strip cache_control from all blocks). Unknown top-level fields:
  drop (log once per field name).
- Model: apply the SAME alias/local-rewrite policy as chat (aliases map
  claude ids → local models; `local/x` prefix works; unmapped TR-namespaced
  claude id with no alias → this request is CLOUD-forced as today unless an
  alias exists — i.e., translation engages only when policy routes LOCAL).

### Response translation (OpenAI → Anthropic), non-streaming
Build MessageResponse: id "msg_"+uuid-ish, type message, role assistant,
model = the ANTHROPIC-visible model id (the inbound one), content blocks:
text → [{type:"text",text}], tool_calls → [{type:"tool_use", id, name,
input: parsed arguments json}], stop_reason mapping (stop→end_turn,
length→max_tokens, tool_calls→tool_use, content_filter→end_turn+log),
usage {input_tokens: prompt_tokens, output_tokens: completion_tokens}.

### Streaming translation (OpenAI SSE → Anthropic SSE event protocol)
Emit the full event dance, spec-faithful:
`message_start` (usage input_tokens if known else 0) → for text:
`content_block_start`(index 0, text) + `content_block_delta`
(text_delta) per chunk + `content_block_stop` → tool calls: one content
block per tool_call index (tool_use start with id/name, then
`input_json_delta` partial_json deltas from the OpenAI
arguments fragments) → `message_delta` {stop_reason, usage.output_tokens
(from the final OpenAI usage chunk — request include_usage on the local
leg like the savings meter does)} → `message_stop`. Every frame is
`event: <type>\ndata: {json}\n\n`. Interleave `ping` events never required —
omit. Text and tool blocks may interleave: track OpenAI tool_call index →
anthropic block index mapping; close a block when another starts or the
stream ends.
