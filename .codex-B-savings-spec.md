## Part 1 — savings meter (the screenshot number, kept HONEST)

### Usage capture
- Non-streaming local responses are already fully buffered (bursty-block
  injection): parse `usage` {prompt_tokens, completion_tokens} from the JSON.
- Streaming local: inject `stream_options:{"include_usage":true}` into the
  LOCAL body only (raw-splice; never into TRBody). While streaming bytes
  through (do NOT slow the hot path), scan only `data: ` lines containing the
  substring `"usage"` and parse just those to capture the final usage chunk.
  If a stream ends with no usage seen, count nothing and increment
  `usage_unknown_total`.
- Cloud (burst/forced-TR) responses: same capture (TR always includes usage;
  final chunk when streaming) — feeds cloud_spend.

### Pricing (labeled counterfactual — this is load-bearing)
Price LOCAL usage at the TR rate of, in precedence order:
1. the alias KEY (the cloud id the user configured the local model to stand
   in for) when the request came through an alias;
2. the requested model when it is TR-namespaced/known in the cached catalog;
3. `-savings-reference <model-id>` (env BURSTY_SAVINGS_REFERENCE) if set;
4. otherwise: count tokens only, no dollars (never invent a price).
Catalog pricing comes from the existing cached control-plane catalog
(prompt/completion per-Mtok fields). Record WHICH reference priced each
request (aggregate counts per reference in stats). Cloud spend is priced at
the actual model returned by the cloud upstream (TR catalog; unknown → count
tokens only under cloud_usage_unpriced).

### Persistence + surfacing
- `~/.bursty/state.json` (respect $XDG_STATE_HOME; `-state-file` override;
  `-state-file=""` disables): atomic tmp+rename writes, flushed every 30s
  when dirty and on clean shutdown. Fields: since, saved_usd_micro,
  local_prompt_tokens, local_completion_tokens, cloud_spend_usd_micro,
  cloud_prompt_tokens, cloud_completion_tokens, per-utc-day cloud spend map
  (pruned to 7 days, used by the budget below). Corrupt/missing file → start
  fresh (log once).
- /stats gains `savings`: {saved_usd, local_tokens_{prompt,completion},
  cloud_spend_usd, usage_unknown_total, references: {<model-id>: count},
  since}. Percentages: local_share of requests.
- Response header `X-Bursty-Saved-USD: <cumulative>` set alongside the
  existing X-Bursty-* headers (cheap: one atomic read).
- Log line at startup (loaded totals), every 30 min, and on shutdown:
  `bursty savings: served NN% locally, saved $X.XX (ref: <model>), cloud spend $Y.YY`.

## Part 2 — cloud egress: loud, capped, killable

- `-cloud=auto|explicit|off` (env BURSTY_CLOUD, default auto):
  auto = today's behavior; explicit = NO automatic bursts (full → 429,
  error → surface local error) — only requests that explicitly target the
  cloud (provider.only/order naming non-local providers) go out; off = the
  cloud upstream is disabled entirely — explicit-cloud requests get the
  fail-closed envelope ("cloud disabled by -cloud=off"), local-capable
  requests behave local-only.
- SIGHUP toggles between the configured mode and `off` at runtime, with a
  LOUD log line each way ("cloud egress DISABLED via SIGHUP" / restored).
  /stats reports the current effective mode.
- `-max-cloud-spend <usd>/day` (env BURSTY_MAX_CLOUD_SPEND, 0=unlimited):
  per-UTC-day cloud spend (from the meter's pricing) exceeding the cap
  blocks ALL cloud sends — burst attempts degrade like -cloud=explicit-
  blocked (429 + Retry-After: seconds-to-UTC-midnight, envelope type
  cloud_budget_exhausted), explicit-cloud requests get the same 429. First
  block of the day logs loudly. Unpriced cloud usage counts $0 toward the
  cap (document this honestly).
- Per-burst structured log at every cloud send completion:
  `bursty cloud: reason=<burst-full|burst-error|forced|...> model=<id>
  prompt_toks=N completion_toks=M est_cost=$Z.ZZZZ` (est_cost omitted when
  unpriced).
- Stats: cloud_blocked_budget, cloud_blocked_mode counters.
